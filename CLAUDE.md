## Project Overview

VS-QA is a log viewer application built with Kotlin Multiplatform (JVM desktop + Android) and Compose Multiplatform. It parses, displays, and navigates Android log files with ProGuard retracing support.

## Prerequisites

The [vs-core](https://github.com/VladislavSumin/vs-core/) repository must be cloned one directory level up from this repo (`../vs-core`). It provides shared build scripts, convention plugins, and core libraries. Settings files reference it via `includeBuild("../vs-core")`.

## Build & Run Commands

```bash
# Run the desktop app (optionally pass log file and mapping file paths)
./gradlew :app:jvmRun [--args "<path_to_log_file> [path_to_mapping_file]"]

# Build desktop fat JAR
./gradlew :app:buildFatJarMain

# Build minimized (ProGuard-shrunk) desktop JAR
./gradlew :app:buildFatJarMainMin

# Build Android APK
./gradlew :app:assembleRelease

# Run all tests
./gradlew allTests

# Run detekt (linting)
./gradlew detekt
```

ProGuard is configured for shrinking only (`-dontobfuscate -dontoptimize`). The minimized JAR task requires a JDK with jmods (not the Studio-bundled JDK).

## Architecture

### Module Structure

- **app** — Composition root. Wires all DI modules, contains platform entry points (`Main.kt` for JVM, `App.kt` for Android).
- **core/** — Shared utility modules (adb-client, boyer-moore-search, proguard-parser, search-utils, utils) and UI modules (design-system, hotkey-controller, file-picker, drag-and-drop).
- **feature/** — Feature modules following a strict api/impl split (see below).
- **build-logic/** — Gradle convention plugins that enforce module patterns.

### Feature api/impl Pattern

Every feature has two Gradle modules: `:feature:<name>:api` and `:feature:<name>:impl`.

**api** contains: Factory interfaces, screen params (serializable `ScreenParams`/`IntentScreenParams`), component interfaces, and interactor interfaces exposed to other features.

**impl** contains: DI module (`Di.kt`), `NavigationRegistrarImpl`, screen/component implementations with `@GenerateFactory`/`@GenerateScreenFactory` annotations, ViewModels, and repository implementations.

Dependency rule: impl modules depend on other features' **api** modules only. The app module is the single composition root that depends on all impl modules.

**log-parser** is a special case with a 3-part split: `api` (parser interfaces), `generic` (GenericLogParser), and `anime` (AnimeLogParser with DI module).

### DI (Kodein)

Each feature's impl has a `Di.kt` with a single `DI.Module` extension on `Modules`:
```kotlin
fun Modules.featureLogViewer() = DI.Module("feature-logViewer") { ... }
```

The composition root is `app/src/commonMain/.../Di.kt` — `createDi()` imports all core and feature modules via `importOnce`.

Factory implementations are auto-generated by KSP processors (`@GenerateFactory`, `@GenerateScreenFactory`).

### Navigation

Built on Decompose (Arkivanov) + custom `vs.core.navigation` abstraction:
- **ScreenParams** — serializable screen identifiers (objects for simple screens, data classes for parameterized ones)
- **NavigationRegistrar** — each feature registers its screens into a central NavigationRegistry
- **NavigationHost** — groups screens (e.g., `TabNavigationHost` for tab-based navigation)
- **NavigationViewModel** — base class providing `open(screenParams)` / `close(screenParams)`
- **Screen** — base class extending `ComponentContext` with `viewModel {}` builder and `Render(modifier)` composable

### Convention Plugins (build-logic)

Layered presets for feature modules:
- `preset/core` — base: KMP JVM + Android library
- `preset/feature-api` — extends core, adds serialization + coroutines + logger
- `preset/feature-api-ui` — extends feature-api, adds Decompose + navigation API
- `preset/feature-impl` — extends feature-api, adds DI + factory generator + KSP hack
- `preset/feature-impl-ui` — extends feature-api-ui + feature-impl, adds Compose + UI core deps

Most convention plugins come from vs-core's `build-scripts` project. Local plugins include `ksp-jvm-hack` (wires KSP into commonMain for JVM-only KMP modules) and `noOpPlugin` (classpath workaround).

### UI

Compose Multiplatform with Material3. Theme defined in `core/ui/design-system` (`QaTheme`, `QaColorScheme`). The design system provides custom components (`QaTextField`, `QaIconButton`, `QaToggleIconButton`) and loads theme config from YAML.

## CI

- **detekt** job (ubuntu) — runs static analysis
- **unit-tests** job (macos) — runs all tests
- **build** job (macos) — builds fat JAR, minimized JAR, and Android APK; uploads all as artifacts

All jobs checkout both vs-qa and vs-core, using composite actions from vs-core for Java setup, Gradle caching, and execution.

Releases trigger on `v*` tags, build all three artifacts, create a GitHub release, and notify via MAX chat.

## Code Style

- Detekt config at `config/analyze/detekt.yml`
- Composable function names may start with uppercase (relaxed `FunctionNaming` rule)
- Trailing commas required on both call and declaration sites
- `LongParameterList` and `TooManyFunctions` rules disabled
- Base package: `ru.vladislavsumin`
